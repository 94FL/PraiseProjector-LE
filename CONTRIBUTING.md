## How to Contribute

1. Fork the repository
2. Create a feature branch from `master`
3. Make your changes
4. Run quality checks:
   ```bash
   npm run lint
   npm run format
   npm run check:ts
   ```
5. Test your changes thoroughly
6. Submit a pull request with a clear description of what you changed and why

## Guidelines

- Keep pull requests focused on a single change
- Follow the existing code style and conventions
- Update documentation if your change affects user-facing behavior
- Do not include third-party code without verifying its license compatibility

## Reporting Issues

For bugs and feature requests, please create an issue in the repository.
